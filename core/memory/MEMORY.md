## Doc backend (vault model — local-first)

The doc backend is **markdown-local-first**: the `.md` files are the single
source of truth; an editor (e.g. Obsidian in vault mode over the backend
root) is a **view**. No external sync layer (the Notion pipeline is retired).

- **MOC** — `Home.md`, the vault entry point (template: `templates/vault/`).
- **`[[wiki-links]]` + frontmatter `aliases`** — resolve dash-slugs →
  underscore files, so the Graph view has no phantom nodes.
- **Dataview boards** — `tasks/Board.md` (tasks by project/status) +
  `specs/_specs-index.md` (specs + AC). AC live as a `- [ ] #ac` checklist
  inside each spec; tasks as `- [ ]` lines with inline fields. Replaces a
  separate DB.
- **Infra graph** — `infra/`, auto-generated by `core/scripts/
  infra_graph_render.py` (reuses the `infra_graph.py` engine): one note per
  code-label (I/S/A/H/R/T/N), with `[[code]]` edges to its Related functions
  + artifact stubs for the non-memory files. `infra_graph.py --check` runs
  the orphan/gap/dup audit.

Deploy renders the templates into `${DOCS_ROOT}` (default
`${workspace.root}/.stc-docs`). The full process + guarantees → playbook
§ Doc backend.

## Conventions

- A `[[link]]` in any file means "read that file now for detail".
- Files here are **reference**, not process. Processes live in `skills/`.
- When you edit a memory file, keep the `situation → action` imperative style
  for rules; move explanations/history to a clearly-marked detail section
  (see playbook § Memory-instruction style).
